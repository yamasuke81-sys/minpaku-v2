# /self-review

「同じミス・同じ手作業を繰り返していないか」を点検し、それを**仕組み（コマンド/ルール）に昇格**する提案を出す。
これが「自ら改善する」ループの中核。**提案までが仕事で、適用はユーザー承認後**に行う。

## 入力

- 引数（任意）: 対象期間（既定: 直近2週間）

## 手順

### 1. 繰り返しパターンの検出

```bash
cd C:/Users/yamas/AI_Workspace/minpaku-v2
git log --since="2 weeks ago" --oneline --stat | head -200
```

- **同じファイル/領域を何度も修正**していないか（＝再発バグの温床）
- 同種のコミットメッセージが繰り返されていないか（同じ手作業の反復）

### 2. 既存の仕組みと突き合わせ

- `.claude/commands/` の既存コマンド一覧を確認し、**まだコマンド化されていない反復作業**を洗い出す
- ローカル実行時のみ: memory(MEMORY.md)に同種の落とし穴が複数あれば、CLAUDE.md の常設「落とし穴リスト」に昇格すべきか検討
  （クラウド routine では memory にアクセス不可。その場合は git 履歴と CLAUDE.md のみで判断し、その旨を明記）

### 3. 昇格提案をまとめる

検出した各パターンについて、次のどれかを提案:
- **新しい `/コマンド` 化**（反復する手作業）
- **CLAUDE.md の落とし穴リストに追記**（再発バグ・既知の罠）
- **既存コマンドの改善**（手順の抜け）
- **対応不要**（一過性）

## 出力

| 検出パターン | 根拠（commit等） | 提案 | 優先度 |
|---|---|---|---|

- 提案は**具体的に**（例: 「`/invoice-debug` を作る」「CLAUDE.md に『◯◯は△△に注意』を追記」）
- **コード/設定の変更・コマンド新規作成は、この場では行わない**。ユーザーが承認した項目のみ別途実施
- 何も見つからなければ「直近は繰り返し問題なし」と1行で報告
