# /checklist-edit

チェックリスト画面の改修を、**3面まとめて**行う。
1面だけ直すと表示・挙動が食い違う（過去に実際に不整合が発生）。

## 入力

- 引数: 変更したい内容（例: `写真必須マークの表示を追加` / `完了ボタンの文言変更`）

## 必ず触る3面

| 面 | ファイル | 役割 |
|----|---------|------|
| スタッフ用 | `public/js/pages/my/my-checklist.js` | スタッフが清掃時に入力する画面 |
| 物件設定用 | `public/js/pages/property-checklist.js` | オーナーがテンプレを編集する画面 |
| 公開ヘルパー | `public/guest-checklist.html` | 公開（宿泊者向け等）ヘルパー |

## 手順

### 1. 現状把握（3面とも読む）

変更対象の箇所が3面それぞれでどう実装されているか Read で確認する。
1面にしか無い機能なら、他2面にも同等に入れるべきか判断する。

### 2. 3面に反映

変更内容を3面それぞれに適用する。表示文言・シンボル・挙動が**揃っている**ことを確認。

- 確認ダイアログが必要な箇所はネイティブ confirm/alert/prompt を使わず
  `showConfirm` / `showAlert` / `showPrompt`（app.js 定義済み）を使う
- ユーザー向け文面に英字フィールド名をそのまま出さない（日本語に翻訳）

### 3. アセットバージョン更新（JS変更時は必須）

`my-checklist.js` / `property-checklist.js` を変更したら、
`public/index.html` の該当 `?v=` を全置換で更新（忘れるとキャッシュで旧版配信）。

### 4. 反映漏れチェック

```bash
cd C:/Users/yamas/AI_Workspace/minpaku-v2
```

変更したキーワードで3ファイルを grep し、3面すべてに入っているか最終確認する。

## 出力

- 3面それぞれに何を変更したかを面ごとに列挙
- 意図的に1面だけ変えた場合は、その理由を明記
- デプロイは `/deploy-v2` で別途（このコマンドは編集まで）
