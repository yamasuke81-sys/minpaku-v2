LINE 連携（物件単位）

この物件専用の LINE 公式アカウントを紐付けます。
複数の Bot を登録すると、無料枠 (200通/月) を使い切った際に自動で切り替えられます。

この手順でできること
LINE 公式アカウント(Bot) をこの物件に紐付けて、新規予約/スタッフ確定/清掃完了などを自動で LINE に通知します。
1物件に最大2つの Bot を登録可能。それぞれ月 200通 無料枠があり、合計 400通/月 まで無料で使えます。
チャネル (Bot) を作成する
注意: 2つの管理画面の違い
LINE には似て非なる 2 つの画面があります。必ず下の「開発者コンソール」から作成してください:
  • 開発者コンソール (今回使う): developers.line.biz/console/ — Messaging API 用チャネルを作る画面
  • 公式アカウントマネージャー: manager.line.biz — ここで作ったアカウントは別途「Messaging API有効化」の手順が必要 (STEP 1-B 参照)
  1. https://developers.line.biz/console/ を開いてログイン
  2. プロバイダー一覧から 「長浜清掃G通知」 をクリック (なければ「新規プロバイダー作成」)
  3. 画面上部の 「新規チャネル作成」 ボタン (緑色) をクリック
  4. チャネルの種類 → 「Messaging API」 を選択
  5. 以下の必須項目を入力:
    • 会社・事業者の所在国/地域: 日本
    • チャネル名: 例「the Terrace 長浜 通知」や「○○物件 Bot #2」(分かりやすい名前)
    • チャネル説明: 例「スタッフへの清掃通知用」
    • 大業種 / 小業種: 「個人」「個人 (その他)」でOK
    • メールアドレス: 自分のメール
  6. 利用規約に同意 → 「作成」 ボタン
  7. 作成完了画面が出れば OK
STEP 1-B: 公式アカウントマネージャーで先に作ってしまった場合
症状: manager.line.biz で公式アカウントは作ったのに、開発者コンソールに表示されない。
解決策: Manager 側で Messaging API を有効化して、既存プロバイダーに紐付ける
  1. https://manager.line.biz/ を開き、該当アカウント (例: 「長浜清掃G通知#2」) を選択
  2. 右上の 「設定」(⚙歯車) をクリック
  3. 左メニューから 「Messaging API」 を選択
  4. 「Messaging APIを利用する」 ボタンをクリック
  5. プロバイダー選択画面で 「長浜清掃G通知」 を選択
    ※ 「新しいプロバイダーを作成」ではなく、既存プロバイダーを選ぶこと！
  6. プライバシーポリシー/サービス利用規約は任意 (空欄でOK) → 「次へ」
  7. 確認画面 → 「OK」
  8. 開発者コンソール (developers.line.biz/console) を再読み込み → 「長浜清掃G通知」プロバイダー配下に新しいチャネルが表示される
  9. このチャネルをクリックして STEP 2 に進む
チャネルアクセストークンを取得
  1. 作成したチャネル (Messaging API タグ) をクリックして入る
  2. 上部タブの 「Messaging API設定」 をクリック
  3. ページを一番下までスクロール
  4. 「チャネルアクセストークン (長期)」 セクションで 「発行」 ボタン
    既に発行済みなら「再発行」でもOK
  5. 表示された長い文字列をコピー 📋 (この後フォームに貼る)
  6. 同じページの 「応答メッセージ」 と 「あいさつメッセージ」 は 「編集」→ オフ に (余計な通数消費を防ぐ)
Bot をスタッフLINEグループに招待
  1. 同じ「Messaging API設定」ページの上の方に 「ボットのベーシック ID」 (例: @431aujis) がある → コピー
  2. 同ページで 「あなたのユーザーID」 を確認して QR コードも表示される。 スマホの LINE アプリで:
    • 友だちタブ → 右上の「追加」アイコン → 「QRコード」で QR スキャン
    • または「検索」→ ID 検索で @xxxxx を入力
  3. Bot を友だち追加したら、既存のスタッフLINEグループ を開く
  4. グループ右上のメニュー (三本線) → 「招待」 → Bot を選択して招待
  5. グループのメンバー一覧に Bot が表示されれば成功 ✓
グループ ID を取得
ここが最難関。2つの方法があります。楽な方 (方法B) を推奨。

方法A: Webhook 設定 (少し手間)

  1. 「Messaging API設定」→ 「Webhook URL」 に一時的に https://webhook.site/ で取得できる URL を設定
  2. 「Webhookの利用」を ON
  3. グループで Bot にメンション (例: @{Bot名} test) を送信
  4. webhook.site に届いた JSON の中の "source.groupId" フィールドが Group ID (例: C1234567890abcdef...)
  5. コピーして Webhook 設定は OFF に戻す

方法B: 自分の User ID で代用 (簡単)

  1. Group ID を取得せず、Webアプリ管理者自身の LINE User ID を使う
  2. 「Messaging API設定」の 「あなたのユーザーID」 (U で始まる文字列) をコピー
  3. これをフォームの「グループ ID」欄に貼る → Webアプリ管理者だけに通知が来る (暫定)
  4. 後日、方法A で本当の Group ID に差し替えるか、運用的にこれで OK ならそのまま
この画面のフォームに登録
  1. 上の 「Bot を追加」 ボタンをクリック
  2. Bot #1 の入力欄が表示される
  3. トークン: STEP 2 でコピーしたチャネルアクセストークンを貼り付け
    グループID: STEP 4 でコピーした Group ID (または User ID) を貼り付け
    表示名: 分かりやすい名前 (例: 「長浜 Bot #1」)
  4. 「この物件の LINE 連携を有効にする」を ON
  5. 入力すると自動保存されます (右下に「✓ 保存しました」が出る)
2つ目の Bot を追加 (無料枠拡張)
  1. STEP 1〜3 を繰り返し、同じプロバイダ内に 別のチャネル を新規作成 (名前: 「○○物件 Bot #2」など)
  2. 新しいトークンを取得 → 新 Bot も 同じグループに招待 (1つのグループに2つ Bot が入る形)
  3. この画面で再度「Bot を追加」→ Bot #2 の欄が表示される
  4. トークン: 新しいトークン
    グループID: Bot #1 と同じ Group ID (同じグループに両方 Bot が居るため)
    表示名: 「長浜 Bot #2」など
  5. 配信戦略: 通常は「残枠 fallback」で OK (Bot #1 が枯渇したら Bot #2 へ自動切替)
