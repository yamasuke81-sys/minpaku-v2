# 1物件1 LINE公式アカウント 運用ガイド

## なぜ物件別運用にするか

LINE Messaging API の無料枠は **1チャネルあたり月200通**。
1チャネルを全物件で共用すると物件・スタッフが増えるほど枠を超えやすい。

**物件ごとに別の LINE 公式アカウント (= Messaging API チャネル) を作成**することで、
各物件が独立した200通/月の枠を持てる。

例: 4物件 × 200通 = **月800通** まで無料。

## LINE Developers Console でチャネルを作成する手順

1. https://developers.line.biz/console/ にアクセスしてログイン
2. 既存プロバイダー「長浜清掃G通知」を選択（または新規プロバイダー作成）
3. 「新規チャネル作成」→「Messaging API」を選択
4. チャネル名: `○○物件 通知` のように物件名を含める
5. 必要事項を入力してチャネルを作成
6. 「Messaging API設定」タブを開く
7. 「チャネルアクセストークン（長期）」欄で「発行」をクリック
8. 表示されたトークンをコピーしておく

## グループ ID の取得方法

グループ LINE に通知を送る場合はグループ ID が必要。

### 方法A: ログ確認（推奨）

1. LINE グループを作成し、作成した Bot を招待
2. グループ内で誰かが Bot にメッセージを送る
3. Firebase Console → Functions → ログ で Webhook リクエストを確認
4. `source.groupId` の値が Group ID（`C` で始まる）

### 方法B: 個人 User ID で代用（簡易）

1. LINE Developers Console → Messaging API設定 → 「あなたのユーザーID」を確認
2. その `U` から始まる文字列を「グループ ID / User ID」欄に入力
3. 動作確認後、グループ ID が取得できたら差し替える

## アプリへの設定手順

1. 民泊管理アプリにオーナーとしてログイン
2. 左メニュー「物件管理」→ 対象物件の「編集」ボタンをクリック
3. 編集モーダル下部の「LINE 連携 (物件単位)」セクションへスクロール
4. 「この物件の LINE 連携を有効にする」を ON にする
5. 「チャネルアクセストークン」に手順8でコピーしたトークンを貼り付け
6. 「グループ ID / User ID」に取得した ID を入力
7. 「アカウント表示名」に管理用の名前を入力（例: 長浜民泊 通知Bot）
8. 「保存」ボタンをクリック

## 通知の優先順位（フォールバック）

`sendLineMessageForProperty` 関数の動作:

1. 物件ドキュメントに `lineEnabled=true && lineChannelToken && lineGroupId` があれば**物件専用チャネル**で送信
2. 物件設定がない（または無効）場合は `settings/notifications` の**共通設定**へフォールバック

既存物件の設定を行う前は従来通り共通チャネルで送信されるため、移行は段階的に行える。

## トラブルシューティング

| 症状 | 確認事項 |
|------|----------|
| 通知が届かない | lineEnabled が ON か確認。トークンが正しいか確認（有効期限なし長期トークンを使用すること） |
| 「LINE送信先が設定されていません」エラー | 物件設定も共通設定も未設定の状態。どちらかを設定する |
| グループに届かず個人に届く | グループ ID ではなく User ID が設定されている。グループ ID に差し替える |
| Bot がグループに入れない | Bot の「グループ・複数人チャットへの参加を許可する」設定を ON にする（Messaging API設定タブ） |
| 月200通を超えた | LINE公式サイトで従量課金プランへの移行を検討、またはさらに物件別チャネルを細分化する |
