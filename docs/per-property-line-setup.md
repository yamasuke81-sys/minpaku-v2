# 1物件 複数 LINE公式アカウント 運用ガイド

## なぜ複数チャネル運用にするか

LINE Messaging API の無料枠は **1チャネルあたり月200通**。
1チャネルを全物件で共用すると物件・スタッフが増えるほど枠を超えやすい。

**物件ごとに複数の LINE 公式アカウント (= Messaging API チャネル) を登録**することで、
各チャネルが独立した200通/月の枠を持てる。

例: 1物件 × 2チャネル = **月400通** まで無料（fallback 運用時）。
例: 4物件 × 2チャネル = **月1,600通** まで無料。

## 1物件に2つのBot (最大2チャネル) を登録する方法

### 配信戦略

| 戦略 | 説明 |
|------|------|
| **残枠 fallback** (推奨) | 配列の先頭 Bot を優先使用。無料枠 (200通) を使い切ったら次の Bot へ自動切替。 |
| **交互使用 (roundrobin)** | 日付 % チャネル数 で Bot を交互に使用。枠を均等に消費したい場合に有効。 |

### 複数運用のメリット/デメリット

**メリット**
- 無料枠の実質的な拡張（月200通 → 400通）
- 一方の Bot がエラーを起こしても fallback 可能
- 物件ごとに通知グループを分離できる

**デメリット**
- LINE Developers Console でチャネルを複数管理する手間が増える
- グループ ID の取得作業が Bot ごとに必要

## LINE Developers Console でチャネルを作成する手順

1. https://developers.line.biz/console/ にアクセスしてログイン
2. 既存プロバイダー「長浜清掃G通知」を選択（または新規プロバイダー作成）
3. 「新規チャネル作成」→「Messaging API」を選択
4. チャネル名: `○○物件 通知` のように物件名を含める
5. 必要事項を入力してチャネルを作成
6. 「Messaging API設定」タブを開く
7. 「チャネルアクセストークン（長期）」欄で「発行」をクリック
8. 表示されたトークンをコピーしておく

## グループ ID の取得方法

グループ LINE に通知を送る場合はグループ ID が必要。

### 方法A: ログ確認（推奨）

1. LINE グループを作成し、作成した Bot を招待
2. グループ内で誰かが Bot にメッセージを送る
3. Firebase Console → Functions → ログ で Webhook リクエストを確認
4. `source.groupId` の値が Group ID（`C` で始まる）

### 方法B: 個人 User ID で代用（簡易）

1. LINE Developers Console → Messaging API設定 → 「あなたのユーザーID」を確認
2. その `U` から始まる文字列を「グループ ID / User ID」欄に入力
3. 動作確認後、グループ ID が取得できたら差し替える

## アプリへの設定手順（複数チャネル）

1. 民泊管理アプリにオーナーとしてログイン
2. 左メニュー「物件管理」→ 対象物件の「編集」ボタンをクリック
3. 編集モーダル下部の「LINE 連携 (物件単位)」セクションへスクロール
4. 「この物件の LINE 連携を有効にする」を ON にする
5. 「配信戦略」を選択（推奨: 残枠 fallback）
6. 「Bot を追加」ボタンで Bot #1 の入力欄を追加
7. Bot #1 のトークン / グループID / 表示名を入力
8. 2つ目が必要な場合は再度「Bot を追加」で Bot #2 を追加
9. 「保存」ボタンをクリック

### サンプル設定例

```
[✓] この物件の LINE 連携を有効にする
配信戦略: 残枠 fallback

Bot #1
  トークン: eyJhbGciOiJIUzI1NiJ9.XXXXXXX...  （長浜清掃G通知 #1）
  グループID: C123abc456def789...
  表示名: 長浜民泊 Bot #1

Bot #2
  トークン: eyJhbGciOiJIUzI1NiJ9.YYYYYYY...  （長浜清掃G通知 #2）
  グループID: C987xyz654mno321...
  表示名: 長浜民泊 Bot #2
```

## 通知の優先順位（フォールバック）

`sendLineMessageForProperty` 関数の動作:

1. **lineChannels[] が設定されている場合**
   - `fallback` 戦略: 先頭から順に残枠 (remaining > 0) のチャネルを使用
   - `roundrobin` 戦略: 日付 % チャネル数 で交互使用、失敗時は次を試みる
2. **lineChannels が空で旧単一フィールドがある場合**: `lineChannelToken` + `lineGroupId` を使用（後方互換）
3. **物件設定がない場合**: `settings/notifications` の**共通設定**へフォールバック

既存の旧単一フィールド設定は引き続き動作するため、移行は段階的に行える。

## トラブルシューティング

| 症状 | 確認事項 |
|------|----------|
| 通知が届かない | lineEnabled が ON か確認。Bot #1 の有効スイッチが ON か確認。トークンが正しいか確認（長期トークンを使用すること） |
| 「全チャネルで無料枠枯渇または送信失敗」エラー | 全 Bot の無料枠が 0 通になっている。LINE Developer Console で使用量を確認 |
| 「LINE送信先が設定されていません」エラー | 物件設定も共通設定も未設定の状態。どちらかを設定する |
| グループに届かず個人に届く | グループ ID ではなく User ID が設定されている。グループ ID に差し替える |
| Bot がグループに入れない | Bot の「グループ・複数人チャットへの参加を許可する」設定を ON にする（Messaging API設定タブ） |
| Bot #2 に切り替わらない | Bot #1 の月使用量が 200通 未満の場合 fallback は発動しない（正常動作） |
